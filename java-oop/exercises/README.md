## 評分標準

每個練習都會根據以下標準評分：

### 程式碼品質 (40%)
- 程式碼結構清晰
- 適當的命名規範
- 註解完整

### 功能實作 (40%)
- 需求完整實作
- 邊界條件處理
- 錯誤處理機制

### OOP概念應用 (20%)
- 適當使用OOP原則
- 設計模式應用
- 程式碼可維護性

2. **效能優化**：分析並優化程式效能

4. **設計文檔**：繪製UML類別圖說明設計
5. **重構練習**：改進現有程式碼的設計
